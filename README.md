# 🛒 Grocery Store Data Analysis using SQL

## 📌 Overview
This project analyzes a grocery store dataset using SQL to generate actionable business insights.  
It focuses on understanding customer behavior, product performance, sales trends, supplier contribution, and employee efficiency.

---

## 🎯 Objectives
- Analyze customer purchasing patterns  
- Identify top-performing products  
- Evaluate sales and revenue trends  
- Measure supplier contribution  
- Assess employee performance  

---

## 🗂️ Dataset
The project uses structured CSV data across multiple tables:

- Customers  
- Orders  
- Order Details  
- Products  
- Suppliers  
- Categories  
- Employees  

---

## 🧠 SQL Skills Demonstrated
- Joins (multi-table joins)  
- Aggregations (SUM, AVG, COUNT)  
- GROUP BY and filtering  
- Subqueries  
- CASE statements  
- Primary & Foreign Key relationships  

---

## 🏗️ Database Design
- Relational schema with proper normalization  
- Primary keys for uniqueness  
- Foreign keys for relationships  
- Cascading updates and deletes  

---

## 📊 Key Analysis & Insights

### 👤 Customer Insights
- Identified top customers by total purchase value  
- Analyzed customer spending patterns  

### 📦 Product Performance
- Determined best-selling products by quantity  
- Calculated revenue contribution per product  
- Category-wise product distribution  

### 📈 Sales Trends
- Monthly revenue and order trends  
- Peak order dates identified  
- Weekday vs weekend sales comparison  

### 🏭 Supplier Analysis
- Top suppliers contributing to revenue  
- Supplier-wise product distribution  

### 👨‍💼 Employee Performance
- Employees handling the most orders  
- Sales contribution per employee  
- Average order value processed
